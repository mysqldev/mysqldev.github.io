Undo Log

Posted on 2027-08-17 Edited on 2025-08-17

Undo Log Header

如果一个事务update多次，则可能产生多条undo log record，这组undo log record视为一个undo log。

一个undo log会先写一个undo log header （参见 trx_undo_header_create），记录产生这个undo log的事务信息以及指向该undo log中的undo log record的指针。也就是说undo log = undo log header + undo log records。如果事务既有insert又有update，则会占用两个undo log header。

undo log header结构如下：

相对偏移量 字段 含义
0 TRX_UNDO_TRX_ID 事务ID
8 TRX_UNDO_TRX_NO 事务no，事务的提交序
16 TRX_UNDO_TRX_NO 只在unpdate undo log中有定义。事务执行了某行的标记删除
18 TRX_UNDO_LOG_START 该undo log的第一条record的偏移量
20 TRX_UNDO_FLAGS undo flags：
+ TRX_UNDO_FLAG_XID = 0x01
+ TRX_UNDO_FLAG_GTID = 0x02
+ TRX_UNDO_FLAG_XA_PREPARE_GTID = 0x04
21 TRX_UNDO_DICT_TRANS 事务是dict相关事务
22 TRX_UNDO_TABLE_ID table id
30 TRX_UNDO_NEXT_LOG 本页面内下一个undo log header（一个页面内允许有多个undo log header，其所属的undo log可能属于其他事务）的偏移量。
32 TRX_UNDO_PREV_LOG 本页面内上一个undo log header的偏移量。
34 TRX_UNDO_HISTORY_NODE 如果该undo log加入了history list，这个位置是list node
46 TRX_UNDO_LOG_OLD_HDR_SIZE = 34 + FLST_NODE_SIZE = 46大小

Undo Page页面

如果一个事务产生的undo log record量比较大，导致undo log header所在页面装不下这个事务的所有undo log record，就需要再分配一个新的页面用来存放溢出来的undo log record。

这类undo log页面类型为FIL_PAGE_UNDO_LOG，其格式为：

页内偏移量 字段 说明
38 (通用page header) TRX_UNDO_PAGE_TYPE TRX_UNDO_INSERT或TRX_UNDO_UPDATE之一
40 TRX_UNDO_PAGE_START 存储undo record页面起始位置
42 TRX_UNDO_PAGE_FREE 空闲区域起始位置
44 TRX_UNDO_PAGE_NODE undo page链表节点（大小为2*FLST_NODE_SIZE=12字节）。
56 链表节点结束

页面头总计TRX_UNDO_PAGE_HDR_SIZE=56-38=18字节。

事务新分配页面并加到undo log header链表的逻辑见trx_undo_add_page。

Undo Log Segment (Undo Segment)

至此知，一个事务产生的undo log record可能涉及多个undo page，且这些undo page会串成链。实际上，每个事务会使用两个undo page的链，一个针对普通表、一个针对临时表，二者主要差异是临时表产生的undo不需要持久化。前面说过，逻辑上一组页面称为segment，undo page所在的segment称之为undo log segment。

trx_undo_add_page入参中undo_ptr就是指这样的segment。

Undo Log Segment Header

Undo Log Segment的第一个页面会额外记录该segment的信息，称之为undo log segment header，格式如下：

页内偏移量 字段 含义
56 TRX_UNDO_STATE 回滚段状态
58 TRX_UNDO_LAST_LOG 该回滚段header页面中最后一条undo log header的偏移量（一个header允许存在多个undo log header）。
60 TRX_UNDO_FSEG_HEADER 该回滚段header所在的file segment的header
70 TRX_UNDO_PAGE_LIST [仅在回滚段第一个页面有定义]，该回滚段的页面链表起始节点。关联到其他非第一个页面的TRX_UNDO_PAGE_NODE字段。
86 回滚段header结束位置。

Undo Log Segment状态取值：

  • TRX_UNDO_ACTIVE
  • TRX_UNDO_CACHED
  • TRX_UNDO_TO_FREE
  • TRX_UNDO_TO_PURGE
  • TRX_UNDO_PREPARED_80028
  • TRX_UNDO_PREPARED
  • TRX_UNDO_PREPARED_IN_TC

Undo Log Header Page

结合前面的描述，综合来看，一个Undo Log Segment的第一个页面开头记录的信息包含3种header：

  • 通用页面header = 0~38字节，共TRX_UNDO_PAGE_HDR=38字节
  • Undo Page Header = 38~56字节，共TRX_UNDO_PAGE_HDR_SIZE=18字节
  • Undo log segment header = 56~86字节，共30字节

这之后才是undo log header和undo log record。
